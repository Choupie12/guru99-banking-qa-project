# Test Plan – Guru99 Banking Application

## Objective
The purpose of this testing project is to validate the main functionalities of the Guru99 Banking Application and ensure that the application behaves according to the expected requirements.

## Scope of Testing
The Guru99 Bank will have 2 roles : Manager & Customer

The following modules were tested:

#### Manager : 
- New Customer : A manager can add a new customer
- Edit Customer : A manager can edit details like address, email, telephone of a customer.
- Delete Customer : A manager can delete a customer.
- New Account : A manager can add a new account for an existing customer.
- Edit Account : A manager can add a edit account details for an existing account
- Delete Account : A manager can add a delete an account for a customer.
- Deposit : A manager can deposit money into any account.
- Withdrawal :A manager can withdraw money from any account. 
- Fund Transfer : A manager can transfer funds from any source bank account to destination account
- Change Password : A manager can change password of only his account. He cannot change passwords of his customers
- Balance Enquiry : A manager can view balance of all the customers who come under his supervision
- Mini Statement : A manager can see mini-statement of any account under his supervision
- Customized Statement : A manager can see Customized -statement of any account
- Login & Logout

#### Customer : 
- Balance enquiry : A customer can have multiple bank accounts. He can view balance of his accounts only
- Fund Transfer : A customer can have transfer funds from his “own” account to any destination account.
- Mini Statement : A customer can see mini-statement of only his own accounts
- Customized Statement : A customer can see Customized- statement of only his “own” accounts
- Change Password : A customer can change password of only his account.
- Login & Logout

## Types of Testing Performed
- Functional Testing
- UI Testing
- Regression Testing

## Testing level
- Unit Testing
- Integration Testing
- System Testing

## Testing Approach
Manual testing techniques were used to execute test cases and validate the application functionalities.
Each test case was executed step by step and the actual result was compared with the expected result.
